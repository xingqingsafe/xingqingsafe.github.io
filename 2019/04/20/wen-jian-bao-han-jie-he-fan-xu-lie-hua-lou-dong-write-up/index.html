0x02 反序列化漏洞

分析hint.php代码，我们需要绕过对payload里面字符串的检测，即：

var_dump($_SERVER['REQUEST_URI']);
    $url = parse_url($_SERVER['REQUEST_URI']);
    var_dump($url);
    parse_str($url['query'],$query);
    var_dump($query);
    foreach($query as $value){
        var_dump($value);
        if (preg_match("/flag/",$value)) { 
            die('stop hacking!');
            exit();
        }
    }

使用如下的url：[http://changame.ichunqiu.com////index.php?file=hint.php&payload=],主要是通过host后面////，

来使得上面的正则匹配绕过preg_match("/flag/",$value)，这里$url = parse_url($_SERVER['REQUEST_URI']);,后得到的$url是：string '////ichunqiu.php?file=demo.php&payload=，然后parse_str($url['query'],$query);，就得到query为：boolean false,然后再经过 foreach($query as $value),这时候value已经变成

array (size=0)
  empty

这时候正则匹配已然绕过，payload我们是通过get方法在开始已经获得的。这个时候我们得构造我们的payload了。仔细审计hint.php中的两个类， __wakeup()这个魔术方法在unserialize($payload);的时候创建完对象Handle后就会调用，这个时候它会将类里面的属性置空：

foreach(get_object_vars($this) as $k => $v) {
            $this->$k = null;

        }

这个时候在这个类的生命周期结束的时候，执行析构函数的时候，就不会执行Flag对象的getFlag函数，那么这个如何绕过呢？

我们可以将这个对象的属性个数变成2个，原理是被反序列话的字符串其中对应的对象的属性个数发生变化时，会导致反序列化失败而同时使得__wakeup失效。参考

http://blog.csdn.net/qq_19876131/article/details/52890854

绕过__wakeup后，在对象Handle析构后，会调用handle也就是Flag的一个实例化对象的getFlag方法，从而将flag.php源码通过echo @highlight_file($this->file,true);高亮显示，得到flag。（ps:flag.php是通过访问网址发现响应码为200说明其存在）
所以构造payload如下：

payload=O:6:"Handle":2:s:14:"%00Handle%00handle";O:4:"Flag":3:s:4:"file";s:8:"flag.php";s:5:"token";N;s:10:"token_flag";R:4;}}

0:6指的是对象名长度是6，Handle:2是Handle类里面有两个属性（原本是1但是为了绕过魔术方法_wake_up改成2），然后s:14是说明Handle的特殊属性handle，因为是私有属性所以是前面有一个Handle，0:4是Flag对象，后面不赘余，主要解释下："token";N;指的是token为空，因为在构造函数里面是通过md5生成的，所以这边我们给它置空，s:10:"token_flag";R:4;,这个是token_flag是一个Flag类的属性，且由于$this->token_flag = $this->token = md5(rand(1,10000));,这个赋值从右向左执行，那么token_flag它对应的是token属性的引用，所以是R，4是指的是token这个引用第一次出现在序列化对象的第几个，在这个O:4:"Flag":3:s:4:"file";s:8:"flag.php";s:5:"token";N;s:10:"token_flag";R:4;}中，Flag是第一个，file是第二，flag.php是第三，所以token是第四个，那么R后面就跟着4.

0x03 总结

首先从文件包含漏洞，使用伪协议读取源码，然后解码base64得到php源码，接下来审计源码发现存在反序列化漏洞，然后绕过index.php中对路径中的参数对字符串flag的检测(防止直接通过文件包含得到源码)，然后再绕过_wakeup魔术方法，这个方法主要是在反序列化得到对象后执行，这个方法将会使得handle属性为空，从而使得在析构函数不能执行Flag实例的getflag方法，所以通过增加Handle的属性绕过。
